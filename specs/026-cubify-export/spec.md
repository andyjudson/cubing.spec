# Feature 026 — cubify-image-export

## Summary

Implement `CubeExporter` in `cubify-harness` supporting both **2D net** and **3D rendered** PNG export. SVG is an internal implementation detail of the 2D path, not a public output format. Stickering mask support throughout. Replaces the Playwright-based headless rendering in `cubify-scripts`.

---

## Motivation

Different CFOP stages suit different image styles:

- **Cross / F2L**: 3D images work well — shows layer context, spatial relationships between pieces
- **OLL / PLL**: 2D net images are cleaner — top-face focus, matches the flat diagram convention used in alg sheets and existing cfop-app images

Currently all images go through Playwright + headful Chromium screenshotting TwistyPlayer (~3–5s per image, headed browser required on macOS). The target is a pure JS export path for both styles, matching the visual quality of existing cubing.js-generated images.

---

## Export Modes

The caller chooses style per export — there is no prescribed mapping of CFOP stage to style. Both modes support any stickering preset.

**Typical usage patterns** (not enforced):
- Case sheets (OLL/PLL reference): 2D net — compact, scannable, matches alg sheet conventions
- Practice app / visualiser: 3D rendered — spatial context, more engaging
- Cross/F2L diagrams: 3D — layer relationships easier to read in 3D
- OLL/PLL in-app: either works; 3D is richer, 2D is faster to generate

### 2D Net (PNG)

Standard cube net layout:

```
     [U]
[L] [F] [R] [B]
     [D]
```

- SVG used internally; PNG is the only public output
- No browser, no WebGL
- Visual parity target: match existing OLL/PLL images from `cubify-scripts`

### 3D Rendered (PNG)

Isometric-style matching the harness renderer:

- Uses `CubeRenderer3D` on `OffscreenCanvas` (browser) or headless WebGL (Node.js)
- Same theme and stickering as 2D path
- Returns PNG `Blob` (browser) or `Buffer` (Node.js)

## Public API

```js
import { CubeExporter } from 'cubify';

// 2D net PNG — caller chooses style, no CFOP stage restriction
const png2d = await CubeExporter.toPNG('R U R\' U\'', {
  stickering: 'oll',
  style: '2d',
  size: 256,
  theme: 'modern',
});

// 3D rendered PNG
const png3d = await CubeExporter.toPNG('R U R\' U\'', {
  stickering: 'oll',
  style: '3d',
  size: 256,
  cameraAngle: 'isometric',  // or { azimuth, elevation }
});

// setupAlg resolution
const png = await CubeExporter.toPNG('R U R\' U\'', {
  setupAlg: 'some setup',
  stickering: 'oll',
  style: '2d',
});
```

---

## setupAlg Resolution

```js
displayState = solved.applyAlg(inv(alg)).applyAlg(setupAlg)
```

Matches `experimentalSetupAnchor: 'end'` behaviour from TwistyPlayer.

## Rendering Style Convention

Different CFOP stages suit different image styles — the caller chooses `style` per export:

| CFOP Stage | Preferred Style | Rationale |
|------------|-----------------|-----------|
| Full / Cross | `'3d'` | Shows layer context and spatial relationships |
| F2L | `'3d'` | Layer structure easier to read in 3D |
| OLL | `'2d'` | Top-face focus; matches flat diagram convention |
| PLL | `'2d'` | Top-face focus; matches flat diagram convention |

This is a convention, not a hard rule — both styles support any stickering mask.

## User Stories

**US-001 — 2D top-down PNG**
`CubeExporter.toPNG(algOrState, { style: '2d', ... })` renders a top-down perspective view (U face + top row of side faces as perspective trapezoids). SVG is used internally; PNG is the only public output. Visual parity with existing cubing.js OLL/PLL images.

**US-002 — 3D rendered PNG**
`CubeExporter.toPNG(algOrState, { style: '3d', ... })` renders the existing `CubeRenderer3D` onto an `OffscreenCanvas` in the browser. Returns a PNG data URL. Used for full/cross/F2L image exports.

**US-003 — Node.js validation script**
`demo/export-test.mjs` produces PNGs for Sune (OLL, 2D), T-Perm (PLL, 2D), and a cross case (3D via harness). Validates non-empty output and correct dimensions.

**US-004 — cubify-scripts migration**
`cubify-scripts/lib/renderer.mjs` calls `CubeExporter.toPNG()` directly. Playwright + headful Chromium dependency removed. Per-image generation time target: <200ms.
