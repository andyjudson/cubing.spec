# Implementation Plan: Cubify Agent Skill

**Branch**: `017-cubify-agent-skill` | **Date**: 2026-03-30 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/017-cubify-agent-skill/spec.md`

## Summary

Add a `/cubify` Claude Code agent skill that generates cube state images from algorithm inputs. Accepts a raw alg string, a named case ID, or a JSON file for batch generation. Uses Playwright (headless Chromium, already installed) to render cubing.js TwistyPlayer and capture SVG (2D/OLL/PLL) or PNG (3D/F2L/default) output. Images are written to `~/.claude/tmp/cubify/`. PNG files are resized to 288px via macOS `sips`.

## Technical Context

**Language/Version**: Node.js 24, ESM (`.mjs`), no TypeScript compilation step
**Primary Dependencies**: `playwright` (headless Chromium rendering), `cubing` (already installed in cfop-app, referenced by path)
**Storage**: Output files to `~/.claude/tmp/cubify/`; temp render HTML to `/tmp/cubify-render.html`
**Testing**: Manual invocation; no test framework (local-only tool)
**Target Platform**: macOS (sips dependency; Chromium from cfop-app Playwright install)
**Project Type**: CLI tool + Claude Code agent skill
**Performance Goals**: Single image under 5 seconds (SC-001)
**Constraints**: macOS only (v1); Playwright Chromium must be installed
**Scale/Scope**: Local developer tool; batch up to 57 OLL cases

### Source Code

```text
.claude/
└── commands/
    └── cubify.md            # Claude Code skill definition (system prompt)

cubify-scripts/              # Standalone Node.js CLI (new top-level dir)
├── package.json             # type: module, playwright dependency
├── cubify.mjs               # Entry point — arg parsing, orchestration
└── lib/
    ├── renderer.mjs         # Playwright headless rendering (SVG extraction + PNG screenshot)
    ├── lookup.mjs           # Case ID lookup from JSON data files
    └── masks.mjs            # Mask preset constants (ported from cubify-app)
```

**Structure Decision**: New `cubify-scripts/` directory at repo root. Keeps the CLI fully independent from `cfop-app` (no Vite/React) and `cubify-app` (browser-only). No build step — plain ESM run directly with `node cubify.mjs`.

## Rendering Architecture

```
/cubify <input>
    ↓
cubify.md (skill)
    ↓ bash
cubify.mjs (arg parser)
    ↓
lookup.mjs          masks.mjs
    ↓                   ↓
renderer.mjs  ←─────────┘
    ↓
Write temp HTML to /tmp/cubify-render.html
(loads cubing.js from cfop-app/node_modules, mounts TwistyPlayer)
    ↓
Playwright launch → page.goto(file://...) → wait for player
    ↓
2D (SVG)                    3D (PNG)
page.$eval → outerHTML      page.screenshot({ clip: player bounds })
write .svg                  write .png → sips -Z 288
    ↓
Return output path → skill reports to user
```
