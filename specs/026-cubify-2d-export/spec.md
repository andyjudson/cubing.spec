# Feature 026 ‚Äî cubify-image-export

## Summary

Implement `CubeExporter` in `cubify-harness` supporting both **2D net** (SVG/PNG, headless-capable) and **3D rendered** (PNG via WebGL) cube image export. Stickering mask support throughout. Replaces the Playwright-based headless rendering in `cubify-scripts`.

---

## Motivation

Different CFOP stages suit different image styles:

- **Cross / F2L**: 3D images work well ‚Äî shows layer context, spatial relationships between pieces
- **OLL / PLL**: 2D net images are cleaner ‚Äî top-face focus, matches the flat diagram convention used in alg sheets and existing cfop-app images

Currently all images go through Playwright + headful Chromium screenshotting TwistyPlayer (~3‚Äì5s per image, headed browser required on macOS). The target is a pure JS export path for both styles, matching the visual quality of existing cubing.js-generated images.

---

## Export Modes

### 2D Net (SVG / PNG)

Standard cube net layout matching cubing.js/TwistyPlayer 2D output style:

```
     [U]
[L] [F] [R] [B]
     [D]
```

- Pure SVG string ‚Äî no browser, no WebGL, no Playwright
- Sticker colours from theme; gap and rounding from theme
- Stickering mask applied (hidden stickers ‚Üí grey)
- PNG via rasterising the SVG (`OffscreenCanvas` in browser, `node-canvas` in Node.js)

**Visual parity target**: match the flat 2D images currently generated by `cubify-scripts` for OLL/PLL cases. Same colour palette, same proportions, same sticker gap style.

### 3D Rendered (PNG)

Isometric-style 3D perspective matching the harness renderer:

- Uses `CubeRenderer3D` on an `OffscreenCanvas` (browser) or `gl` headless WebGL (Node.js)
- Same theme, same stickering mask
- Suitable for cross/F2L where spatial context matters
- Returns PNG `Blob` (browser) or `Buffer` (Node.js)

## Public API

```js
import { CubeExporter } from 'cubify';

// 2D net
const svg = await CubeExporter.toSVG('R U R\' U\'', {
  stickering: 'oll',
  size: 256,
  theme: 'modern',
});

const png2d = await CubeExporter.toPNG('R U R\' U\'', {
  stickering: 'oll',
  style: '2d',      // default for OLL/PLL
  size: 256,
});

// 3D rendered
const png3d = await CubeExporter.toPNG('R U R\' U\'', {
  stickering: 'f2l',
  style: '3d',      // default for cross/F2L
  size: 256,
  cameraAngle: 'isometric',  // or { azimuth, elevation }
});

// setupAlg (matches TwistyPlayer experimentalSetupAnchor: 'end')
const svg = await CubeExporter.toSVG('R U R\' U\'', {
  setupAlg: 'some setup',
  stickering: 'oll',
});
```

## setupAlg Resolution

```js
displayState = solved.applyAlg(inv(alg)).applyAlg(setupAlg)
```

Matches `experimentalSetupAnchor: 'end'` behaviour from TwistyPlayer.

## User Stories

**US-001 ‚Äî SVG cube net**
`CubeRenderer2D.toSVG(state, options)` returns a valid SVG string. Visual parity with existing cubing.js 2D output.

**US-002 ‚Äî 2D PNG export**
`CubeExporter.toPNG(algOrState, { style: '2d', ... })` rasterises SVG. Works headlessly in Node.js.

**US-003 ‚Äî 3D PNG export (browser)**
`CubeExporter.toPNG(algOrState, { style: '3d', ... })` renders via `OffscreenCanvas` + Three.js. Returns `Blob`.

**US-004 ‚Äî Node.js validation script**
`demo/export-test.mjs` produces SVG and PNG for Sune (OLL), T-Perm (PLL), and a cross case. Validates sticker count and non-empty output.

**US-005 ‚Äî cubify-scripts migration**
`cubify-scripts/lib/renderer.mjs` calls `CubeExporter` directly. Playwright + headful Chromium dependency removed. Per-image generation time target: <200ms.
