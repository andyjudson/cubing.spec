Patch 0007: Make Beginner page the default landing route

Problem
-------
The app currently navigates to the Intuitive page on initial load. For new users, the Beginner/primer page is a better default entry to explain basics and orient users.

Goal
----
Change the app's default route to the Beginner/primer page while preserving deep-linking behavior and navigation order.

Proposed Changes
----------------
1. Routing: Update the router configuration (likely `src/main.tsx`, `src/App.tsx`, or the routing setup file) so the root path (`/` or app base) loads the Beginner/primer page component instead of the Intuitive page.

2. Navigation: Update the navbar or any `NavLink` ordering if needed so the Beginner link is primary; ensure `aria-current` semantics remain correct.

3. Deep links: Ensure pages with direct links (e.g., `/intuitive`) still resolve correctly; maintain backward compatibility for saved links.

4. Tests: Manual smoke test for initial load, navigation to beginner, and deep links to intuitive/notation pages.

Notes
-----
- If the app uses server-side redirects or an index.html rewrite, ensure the routing change does not conflict with deployed hosting rules.
- Keep the change small and reversible; document in changelog and the spec notes.
