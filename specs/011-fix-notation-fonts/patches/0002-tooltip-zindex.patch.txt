# Patch proposal: tooltip z-index / stacking context (BGR page)
# Target: components/AlgorithmCard.css and page-level containers

Problem:
- Tooltip elements are appearing behind other content on the BGR page (z-index/stacking context issue).

Proposal:
- Ensure tooltip container uses a high z-index and is placed in a stacking context that isn't clipped by ancestor overflow rules.

*** Begin Patch
*** Update File: cfop-app/src/components/AlgorithmCard.css
@@
 .algo-card .tooltip {
-  position: absolute;
+  position: absolute;
   left: calc(100% + 12px);
   top: 50%;
   transform: translateY(-50%);
   background: var(--color-bg-base);
   border: 1px solid var(--color-border-medium);
   border-radius: var(--radius-md);
   padding: var(--space-md) var(--space-lg);
   box-shadow: var(--shadow-xl);
   z-index: 100;
   min-width: 200px;
   max-width: 280px;
@@
 }
+
+/* Ensure tooltip is rendered above modals/other cards when needed */
+.algo-card .tooltip.is-topmost {
+  z-index: 9999;
+  position: fixed; /* fallback placement strategy to escape overflow: hidden parents */
+}
*** End Patch

Implementation notes:
- Adding a `is-topmost` modifier allows JS to toggle placement when the tooltip is being shown (compute position and toggle fixed placement).
- If ancestor elements use `overflow: hidden`, absolute-positioned tooltip will be clipped; using `position: fixed` and computing coordinates in JS avoids that.
- Alternatively, render tooltips into a top-level portal element (`#tooltip-portal`) to avoid stacking/overflow issues.
