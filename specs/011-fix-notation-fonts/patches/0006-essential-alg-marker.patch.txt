Patch 0006: Restore essential algorithm marker (star)

Problem
-------
The "essential" algorithm marker (star icon) that indicates important/favorited algorithms is not rendering on algorithm cards or notation grids. This reduces discoverability and breaks the expected UX.

Goal
----
Restore the star icon, its positioning, hover/tap target, and toggle behavior. Ensure the marker is accessible (aria-pressed or aria-label) and respects design tokens for color and spacing.

Proposed Fix (CSS + small JS/TSX change)
---------------------------------------
1. CSS: add a `.algo-card .essential-marker` element styled to sit in the top-right of the card, above other card content. Use `position: absolute` or `position: fixed` depending on layout, and ensure high `z-index`.

.example CSS:
.algo-card { position: relative; }
.algo-card .essential-marker {
  position: absolute;
  top: 8px;
  right: 8px;
  width: 28px;
  height: 28px;
  display: inline-flex;
  align-items: center;
  justify-content: center;
  color: var(--color-text-tertiary);
  background: transparent;
  border-radius: 6px;
  z-index: 2000;
}
.algo-card .essential-marker.active { color: var(--color-accent-primary); }

2. TSX: ensure `AlgorithmCard` renders the marker when `props.isEssential` (or computed) is true, and wire a click handler `onToggleEssential` that toggles state and persists (localStorage or API as appropriate). Add ARIA attributes:

<button className={`essential-marker ${isEssential? 'active':''}`} aria-pressed={isEssential} aria-label={isEssential? 'Unmark essential' : 'Mark essential'} onClick={toggle}>★</button>

3. Tests: Manual QA to toggle marker on/off, verify keyboard focus, verify marker persists per chosen storage, and verify screen reader announces state.

Notes
-----
- If cards are used in compact/inline contexts where `position: absolute` won't work, consider rendering marker inline with `float: right` fallback or a variant class.
- Keep styling consistent with design tokens (`--space-*`, `--color-accent-primary`, `--radius-sm`).
